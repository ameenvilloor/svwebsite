73 // Some elementary notes on abstract base classes (ABC's)
74 // ------------------------------------------------------
75 // ABC's provide a means for defining an *interface*. Since (by
76 // definition) they contain pure virtual methods, objects of these
77 // classes can not be instantiated. Clients of ABC's are interested
78 // in using the abstract interface, but can not work with the objects
79 // themselves. Instead, clients instantiate concrete classes derived
80 // from the ABC. And then, to use the abstraction, clients use
81 // *pointers* or *references* to the ABC. See Meyers' Effective C++,
82 // Item 34.
