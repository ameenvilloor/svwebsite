59 // Some elementary notes on abstract base classes (ABC's)
60 // ------------------------------------------------------
61 // ABC's provide a means for defining an *interface*. Since (by
62 // definition) they contain pure methods, objects of these
63 // classes can not be instantiated. Clients of ABC's are interested
64 // in using the abstract interface, but can not work with the objects
65 // themselves. Instead, clients instantiate concrete classes derived
66 // from the ABC. And then, to use the abstraction, clients use
67 // *pointers* or *references* to the ABC. See Meyers' Effective C++,
68 // Item 34.
69 //
