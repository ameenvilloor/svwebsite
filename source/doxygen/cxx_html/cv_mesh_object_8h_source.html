54 // Some elementary notes on abstract base classes (ABC's)
55 // ------------------------------------------------------
56 // ABC's provide a means for defining an *interface*. Since (by
57 // definition) they contain pure virtual methods, objects of these
58 // classes can not be instantiated. Clients of ABC's are interested
59 // in using the abstract interface, but can not work with the objects
60 // themselves. Instead, clients instantiate concrete classes derived
61 // from the ABC. And then, to use the abstraction, clients use
62 // *pointers* or *references* to the ABC. See Meyers' Effective C++,
63 // Item 34.
