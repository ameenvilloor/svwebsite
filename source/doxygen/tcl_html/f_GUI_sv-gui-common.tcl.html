geom_createRatioMap (inlet_mesh_face radmax result)

by Nathan Wilson
This proc maps the scalars given on an input PolyData object to a destination PolyData object and returns a new PolyData. This routine is not general and makes numerous assumptions about the nature of each point set. See notes.
Notes: This code does a simple mapping of the Womersley analytic profile on to the points of the the output PolyData. This mapping assumes that the objects are similar in that if you send a ray from the center of each object the relationship between inner (r) and outer (R) radius is given by r_out = r_in * R_out / R_in for any given angle. We also scale the result scalars assuming they represent a a through plane flux so that the input and output PolyData's have the same through plane flux. This code requires that both PolyData's be in the z=0 plane. Each should consist of a single region. Iso-parametric interpolation functions are used to evaluate the scalar values and for calculating through plane flow rate. The absolute value of the ratio is the radius ratio, where positive values denote interior and negative values denote boundary nodes. This code would be much faster if it were rewritten in C.
Argument: inlet_mesh_face PolyData onto which to map the scalar data.
Argument: radmax not documented
Argument: result Name of new repository PolyData object to be created.
Returns: status
